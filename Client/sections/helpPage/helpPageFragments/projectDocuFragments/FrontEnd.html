Front End in General

File Structure and important Files

Folder/File Name Purpose
Folder "Client" All client related files can be found inside this folder. The structure inside this folder is further separated into Assets, Sections and JavaScript files
Folder "Assets" Inside this folder are all CSS files, changing the style of the html elements, as well as resource folder, containing images and icons, used in th web app
Folder "Sections" The sections folder included all html fragments, which are inserted into the main index.html file to enable a "one page feeling" The section folder is further separated into different view folders, where the actual html data fragments can be found.
File "app.js" Main Angular JS module, which lists also needed dependencies for the client web application
File "app.routes.js" This file consists of the routing system of the web app. It routes the user to different html fragments, depending on the request.
File "app.service.js" This file is responsible for offering controllers access to a variety of "services". These services are operating globally. Controllers are sharing these services.
File "index.html" The main HTML page, which defines the main elements of the website. It also defines all libraries and files, needed for the web application
File "main.ctrl.js" This file consists of all controllers, necessary for the front end. The front end logic is written inside this JS file.

Layout

The front End was built, based on the AngularJS Framework. Additionally, the design is a combination of AngularJS material and Bootstrap.

Landing Page

The landing page consists of 2 buttons, which directs users either to the bar chart view or the circle view.

Circle View

The circle view takes data from the back-end and draws, depending on the amount of carriers, circles on the page. This is done with HTML5 Canvas.

Furthermore, it:

  • allows users to select circles.
  • allows users to compare selected carriers, in different graphs.
  • allows users to refresh the page and displays the last refresh date.
  • displays the current data used.
Bar Chart View

The bar chart view takes data from the back-end and plots, depending on the amount of carriers,a bar chart on the page. This is done with the library Chart.js.

Furthermore, it:

  • allows users to switch in different graphs.
  • allows users to refresh the page and displays the last refresh date.
  • displays the current data used.
Continuous Consumption Graph

The Continuous Consumption Graph takes data from the back-end and plots, depending on the amount of carriers,a line chart on the page. This is done with the library Dygraph. The y-Axis of the graph is by default energy, while the x-Axis is always time.

Furthermore, it:

  • allows users to select different carriers to be compared.
  • allows users to select different iterations to be compared.
  • allows users to select a different dimension to be displayed.
  • allows users to select different Files to be displayed.
  • allows users to refresh the page and displays the last refresh date.
  • allows users to switch to different graphs, while keeping the options(as far as it makes sense).
  • displays the current options used by the graph for plotting.
Average Consumption Graph

The Average Consumption Graph takes data from the back-end and plots, depending on the amount of carriers,a line chart on the page. This is done with the library Dygraph. The y-Axis of the graph is by default energy, while the x-Axis is always iteration.

Furthermore, it:

  • allows users to select different carriers to be compared.
  • allows users to select different iterations to be compared.
  • allows users to select a different dimension to be displayed.

Flexibility Graph

The Flexibility Graph takes data from the back-end and plots, depending on the amount of carriers,a line chart on the page. This is done with the library Dygraph. The y-Axis of the graph is always speed, while the x-Axis is always absolute time.

Furthermore, it:

  • allows users to select different carriers to be compared.
  • allows users to select different iterations to be compared.
  • allows users to select different Files to be displayed.
  • allows users to refresh the page and displays the last refresh date.
  • allows users to switch to different graphs, while keeping the options(as far as it makes sense).
  • displays the current options used by the graph for plotting.
  • displays flexibility between multiple carriers in the form of a flexibility KPI.
Energy Spike Table

The Energy Spike Table shows all carriers with their respective Iteration of a selected file in a table. The first row shows the carrier and iteration combination that has the highest energy spike in the file.

Furthermore, it:

  • allows users to refresh the page and displays the last refresh date.
  • allows users to investigate the data set by switching to the continuous energy graph.
Simulation Page

The Simulation Page allows users to upload new data files into the web application and start a new simulation, by selecting a file to be used by all graphs. The forms are built with Bootstrap.

Furthermore, it:

  • allows users to start a new simulation, based on different parameters.
  • allows users to reset the system.
Help Page

The Help Page is built with Bootstrap-UI Tabs structure. The tabs consist of Glossary, Manual and Project Documentation. The Content of the tab was also designed with Bootstrap and AngularJS functionality.
